# Web Dashboard

Neptune includes a Next.js web dashboard for managing analysis jobs through a browser interface.

## Features

- **GitHub OAuth** authentication via Better Auth
- **Job management** — Upload binaries, track progress, view results
- **Real-time progress** — SSE-powered live updates during analysis
- **Signature browser** — Search and browse the signature database
- **Report viewer** — View HTML reports and download SBOMs
- **Malware triage** — Upload samples for YARA + capa analysis, view signal rankings, behavior hypotheses, and case results

## Setup

### Prerequisites

- Node.js 18+ or [Bun](https://bun.sh)
- Running Neptune API server (`neptune serve`)

### Environment variables

```bash
NEPTUNE_API_URL=http://localhost:9090
BETTER_AUTH_SECRET=your-secret-here
NEXT_PUBLIC_APP_URL=http://localhost:3000
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
```

### Run

```bash
# Development
bun run dev:web

# Or standalone
cd web
bun install
bun run dev
```

The dashboard runs on `http://localhost:3000` by default.

## GitHub OAuth setup

1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
2. Create a new OAuth App
3. Set the callback URL to `http://localhost:3000/api/auth/callback/github` (or your production URL)
4. Copy the Client ID and Client Secret to your environment variables
