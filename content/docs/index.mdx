# Neptune

Neptune analyzes stripped firmware binaries to identify statically-linked C/C++ libraries, then generates regulatory-compliant SBOMs and correlates known vulnerabilities.

Regulations like **US Executive Order 14028**, the **EU Cyber Resilience Act**, and **FDA 524B** now require Software Bills of Materials for firmware, but stripped binaries ship without manifests. Neptune solves this.

## How it works

Neptune combines multiple identification techniques with confidence-weighted scoring:

1. **Multi-format extraction** — Parse ELF, PE, and Mach-O binaries with entropy analysis and section mapping
2. **Multi-architecture disassembly** — Capstone-backed disassembly for x86, x86_64, ARM, AArch64, MIPS, RISC-V, and PowerPC
3. **Layered identification** — Strings, constants, compiler ID, signature fingerprinting, cross-correlation, and AI analysis each contribute scored evidence
4. **SBOM generation** — CycloneDX 1.6, SPDX 2.3, VEX, HTML reports, and JSON output
5. **Vulnerability correlation** — Cross-reference identified components against NVD (via CPE) and OSV (via PURL)

## Supported formats

| Category | Supported |
|---|---|
| **Binary formats** | ELF, PE, Mach-O |
| **Architectures** | x86, x86_64, ARM, AArch64, MIPS, RISC-V, PowerPC |
| **SBOM formats** | CycloneDX 1.6, SPDX 2.3, VEX, HTML, JSON |
| **Firmware containers** | SquashFS, JFFS2, CramFS, CPIO, UBI, Android Boot |
| **Vuln databases** | NVD (CPE), OSV (PURL) |
